# Feature Specification: 个人资料编辑功能

**Created**: 2026-07-10  
**Status**: Draft  
**Input**: User description: "目前个人页面中的资料编辑功能还没有实现。请你实现头像编辑（从图库上传）、昵称编辑、性别、出生日期、身高、体重编辑功能。"

## Overview

在个人（Mine）页面中实现完整的资料编辑功能。用户可点击"个人资料"进入资料列表页，查看并编辑头像（从图库选择）、昵称（文本输入）、性别（男/女选择）、出生日期（日期选择器）、身高（滑轮选择器 100-250cm）、体重（滑轮选择器 20-300kg）。每个字段点击后跳转到独立编辑页面进行修改。Mine 页面顶部的用户信息区实时显示当前头像和昵称。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 进入个人资料页面查看当前资料 (Priority: P1)

用户在 Mine 页面点击"个人资料"行，导航到个人资料列表页，看到所有字段的当前值（头像、昵称、性别、出生日期、身高、体重），未设置的字段显示"未设置"占位文本。

**Why this priority**: 这是所有编辑操作的前提——用户必须先能看到资料页面。

**Independent Test**: 打开应用→切换到 Mine 页→点击"个人资料"→页面展示6个字段行，未设置的显示占位文本。

**Acceptance Scenarios**:

1. **Given** 用户首次打开应用（无任何个人资料数据），**When** 点击"个人资料"，**Then** 显示6个字段行：头像（默认占位图标）、昵称（"未设置"）、性别（"未设置"）、出生日期（"未设置"）、身高（"未设置"）、体重（"未设置"），每行右侧有右箭头
2. **Given** 用户已设置过昵称为"小明"，**When** 点击"个人资料"，**Then** 昵称行显示"小明"而非"未设置"

---

### User Story 2 - 编辑头像 (Priority: P1)

用户在个人资料页点击头像行，跳转到头像编辑页，点击"从图库选择"后打开系统图库，选择一张图片后直接作为头像使用（不裁剪），返回资料页后头像已更新，Mine 页面用户信息区同步显示新头像。

**Why this priority**: 头像是最直观的个人信息展示，与昵称同等重要。

**Independent Test**: 进入资料页→点击头像行→选择图片→返回→头像已更新。

**Acceptance Scenarios**:

1. **Given** 用户在资料页，头像为默认占位图标，**When** 点击头像行跳转到头像编辑页，点击"从图库选择"，选择一张图片，**Then** 头像立即更新为所选图片，返回资料页和 Mine 页均显示新头像
2. **Given** 用户在头像编辑页，**When** 点击返回不选择图片，**Then** 头像保持不变

---

### User Story 3 - 编辑昵称 (Priority: P1)

用户在个人资料页点击昵称行，跳转到昵称编辑页，输入新昵称后点击保存，返回资料页后昵称已更新，Mine 页面用户信息区同步显示新昵称。

**Why this priority**: 昵称是用户身份的核心标识。

**Independent Test**: 进入资料页→点击昵称行→输入"小明"→保存→昵称已更新。

**Acceptance Scenarios**:

1. **Given** 用户在资料页，昵称为"未设置"，**When** 点击昵称行跳转编辑页，输入"小明"并保存，**Then** 返回后昵称显示"小明"，Mine 页用户名也显示"小明"
2. **Given** 用户在昵称编辑页，**When** 未输入任何内容直接保存，**Then** 昵称保持原值不变（不允许空昵称）

---

### User Story 4 - 编辑性别 (Priority: P2)

用户在个人资料页点击性别行，跳转到性别选择页，展示"男"和"女"两个选项，选中后自动保存并返回资料页。

**Why this priority**: 性别是基本个人信息，但优先级略低于头像和昵称。

**Independent Test**: 进入资料页→点击性别行→选择"男"→返回→性别显示"男"。

**Acceptance Scenarios**:

1. **Given** 用户在资料页，性别为"未设置"，**When** 点击性别行跳转选择页，点击"男"，**Then** 自动保存并返回，性别显示"男"
2. **Given** 用户性别已设为"男"，**When** 再次点击性别行选择"女"，**Then** 性别更新为"女"

---

### User Story 5 - 编辑出生日期 (Priority: P2)

用户在个人资料页点击出生日期行，跳转到日期编辑页，弹出日期滑轮选择器，选择日期后保存并返回资料页。

**Why this priority**: 出生日期是常见个人信息，但使用频率较低。

**Independent Test**: 进入资料页→点击出生日期行→选择2000年1月1日→保存→出生日期显示"2000-01-01"。

**Acceptance Scenarios**:

1. **Given** 用户在资料页，出生日期为"未设置"，**When** 点击出生日期行跳转编辑页，选择日期后保存，**Then** 返回后出生日期显示为"YYYY-MM-DD"格式
2. **Given** 用户在日期编辑页，**When** 选择未来日期，**Then** 允许选择（不限制范围上限为今天）

---

### User Story 6 - 编辑身高 (Priority: P2)

用户在个人资料页点击身高行，跳转到身高编辑页，展示滑轮选择器（范围100-250cm），选择后保存并返回资料页，身高显示带单位"cm"。

**Why this priority**: 身高是健康应用的基础数据，与体重同等重要。

**Independent Test**: 进入资料页→点击身高行→滑动选择175→保存→身高显示"175cm"。

**Acceptance Scenarios**:

1. **Given** 用户在资料页，身高为"未设置"，**When** 点击身高行跳转编辑页，滑动选择175并保存，**Then** 返回后身高显示"175cm"
2. **Given** 用户在身高编辑页，**When** 默认选中值应为170cm（中间值），**Then** 选择器初始位置在170附近

---

### User Story 7 - 编辑体重 (Priority: P2)

用户在个人资料页点击体重行，跳转到体重编辑页，展示滑轮选择器（范围20-300kg，精度0.1kg），选择后保存并返回资料页，体重显示带单位"kg"。

**Why this priority**: 体重是健康应用的基础数据。

**Independent Test**: 进入资料页→点击体重行→滑动选择65.5→保存→体重显示"65.5kg"。

**Acceptance Scenarios**:

1. **Given** 用户在资料页，体重为"未设置"，**When** 点击体重行跳转编辑页，滑动选择65.5并保存，**Then** 返回后体重显示"65.5kg"
2. **Given** 用户在体重编辑页，**When** 默认选中值应为65.0kg（中间值），**Then** 选择器初始位置在65.0附近

---

### Edge Cases

- 用户拒绝图库权限时的处理：显示提示，头像不变
- 用户在选择器/编辑页点击返回但不保存：数据不变
- 昵称为空时保存：不允许，保持原值
- 所有数据持久化后重启应用：资料不丢失
- 头像 URI 在应用重启后是否仍有效（沙箱路径持久性问题）

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: Mine 页面"个人资料"行必须有 onClick 处理，点击后导航到个人资料列表页
- **FR-002**: 个人资料列表页展示6个字段：头像、昵称、性别、出生日期、身高、体重，每行显示标签+当前值+右箭头
- **FR-003**: 每个字段行点击后跳转到独立的编辑页面
- **FR-004**: 头像编辑页提供"从图库选择"按钮，调用系统 PhotoViewPicker 选择图片，不裁剪直接使用
- **FR-005**: 昵称编辑页提供文本输入框和保存按钮，不允许空昵称
- **FR-006**: 性别选择页提供"男"和"女"两个选项，选中后自动保存并返回
- **FR-007**: 出生日期编辑页提供日期滑轮选择器，格式 YYYY-MM-DD
- **FR-008**: 身高编辑页提供数字滑轮选择器，范围 100-250cm，步进1cm
- **FR-009**: 体重编辑页提供数字滑轮选择器，范围 20-300kg，步进0.1kg
- **FR-010**: 所有资料数据必须持久化存储，应用重启后数据不丢失
- **FR-011**: Mine 页面用户信息区（UserInfoComponent）实时显示当前头像和昵称
- **FR-012**: 未设置的字段在资料列表页显示"未设置"占位文本
- **FR-013**: 资料列表页提供返回按钮导航回 Mine 页

### Key Entities

- **UserProfile**: 用户资料实体，字段包括：avatarUri（头像图片URI，string）、nickname（昵称，string）、gender（性别，0=未设置/1=男/2=女）、birthday（出生日期，string YYYY-MM-DD）、height（身高，number cm）、weight（体重，number kg）

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户能在3次点击内进入任意字段的编辑页面
- **SC-002**: 编辑任意字段保存后，资料列表页和 Mine 页用户信息区在1秒内同步更新
- **SC-003**: 应用重启后所有已保存的资料数据完整恢复
- **SC-004**: 6个字段的编辑功能均可独立正常工作，互不影响

## Assumptions

- 使用 Preferences 键值对存储用户资料（单用户场景，无需 RDB 表）
- 头像图片 URI 使用应用沙箱内的临时路径，选择后将图片复制到应用持久化目录以保证重启后可用
- 头像显示为圆形裁剪（Clip Circle）
- 昵称最大长度限制为20个字符
- 性别只提供"男"和"女"两个选项
- 出生日期格式统一为 YYYY-MM-DD
- 身高范围 100-250cm，步进1cm
- 体重范围 20-300kg，步进0.1kg
- 每个字段的编辑页面使用 Navigation 子页面模式（NavDestination + pushPathByName）
- 编辑页面的 UI 风格与现有 AchievementComponent 等子页面保持一致
