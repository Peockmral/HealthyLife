# Feature Specification: 当天任务进度对话框

**Created**: 2026-07-09  
**Status**: Draft  
**Input**: 用户描述: "点击当天的日历图标，可以弹出一个框，能够显示当天每项任务的进度条。如果是早起、早睡这样的任务，就只有0%和100%两种状态；如果是吃苹果、刷牙这种设置了次数的任务，就按已经完成的次数来设置百分比。"

**确认决策**: ①仅"今天"图标点击后弹框，翻页到其他天点击仅切换日期、不弹框；②每行展示任务图标+任务名+横向进度条+数值文字（次数型如"1/3个"，布尔型完成显示"已完成"）；③采用与现有打卡弹窗一致的居中自定义对话框样式。

## Overview

用户在首页周历上点击表示"今天"的日历图标时，弹出居中对话框，以横向进度条形式展示当天每个已开启任务的完成进度。布尔型任务（早起、早睡）显示 0% 或 100% 两种状态；次数型任务（吃苹果、刷牙等）按已完成次数与目标次数之比显示百分比，并附带数值文字。该功能让用户在周历入口即可快速回看当天各项健康习惯的达成情况，无需逐个进入任务列表。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 点击今天日历图标弹出进度对话框 (Priority: P1)

用户在首页周历上点击表示"今天"且当前已选中的日历图标，系统弹出居中对话框，列出当天每个已开启任务及其进度条。

**Why this priority**: 这是功能的核心入口与展示，独立可测——点击即见对话框弹出，是整个功能的最小可用切片。

**Independent Test**: 在首页点击今天已选中的日历图标，验证居中对话框弹出并列出当天任务项。

**Acceptance Scenarios**:

1. **Given** 今天是周历中当前已选中的那一天，**When** 用户点击今天对应的日历图标，**Then** 弹出居中进度对话框，列出当天所有 isOpen=true 的任务条目。
2. **Given** 用户已翻页到非今天的某一天，**When** 点击该天的日历图标，**Then** 仅切换日期（将该天设为选中），不弹出进度对话框。
3. **Given** 今天未被选中（用户在别页），**When** 点击今天图标，**Then** 先切换为今天并选中，不弹框；再次点击今天已选中图标，**Then** 弹出进度对话框。

---

### User Story 2 - 按任务类型正确展示进度 (Priority: P2)

对话框内每个任务条目按其类型展示进度条与文字：布尔型（早起/早睡）未完成显示 0% 进度、完成显示 100%；次数型（吃苹果等）按已完成次数与目标次数之比显示进度，并附带"已完成次数/目标次数 单位"文字。

**Why this priority**: 进度的正确计算与展示是功能价值的核心，依赖现有数据模型（TaskInfo/DayTaskInfo/TaskBaseInfo），可在 US1 弹框能力之上独立验证各任务进度值。

**Independent Test**: 构造不同完成状态的任务（布尔型已/未打卡、次数型部分完成），打开对话框核对每个条目的进度条比例与数值文字。

**Acceptance Scenarios**:

1. **Given** 早起任务当天未打卡，**When** 打开进度对话框，**Then** 早起条目进度条为 0%，文字显示未完成态（如"--"，与现有列表一致）。
2. **Given** 早起任务当天已打卡，**When** 打开进度对话框，**Then** 早起条目进度条为 100%，文字显示"已完成"。
3. **Given** 吃苹果任务目标3次、已完成1次，**When** 打开进度对话框，**Then** 进度条约 33%，文字显示"1/3个"。
4. **Given** 喝水任务目标1.5升、已完成0.5升，**When** 打开进度对话框，**Then** 进度条约 33%，文字显示"0.5/1.5升"。
5. **Given** 刷牙任务目标1次、已完成1次（已完成），**When** 打开进度对话框，**Then** 进度条 100%，文字显示"已完成"。

---

### User Story 3 - 关闭对话框并恢复正常交互 (Priority: P3)

用户可通过点击对话框外部空白区域或关闭操作关闭对话框，关闭后周历交互恢复正常（可继续切换日期、再次点击今天图标重新打开）。

**Why this priority**: 关闭是基础收尾体验，确保对话框不阻塞主页后续操作。

**Independent Test**: 打开对话框后点击空白处或关闭，验证对话框消失且主页可正常交互。

**Acceptance Scenarios**:

1. **Given** 进度对话框已打开，**When** 用户点击对话框外部空白区域，**Then** 对话框关闭，返回周历主页。
2. **Given** 对话框关闭后，**When** 用户再次点击今天已选中图标，**Then** 能再次弹出反映最新进度的对话框。

---

### Edge Cases

- **当天没有任何已开启任务（isOpen 全为 false）**：对话框弹出但任务列表为空，显示空状态提示（如"今天还没有开启的任务"），不报错。
- **次数型任务 finValue 超过 targetValue**（历史/异常数据）：进度封顶 100%，文字仍按实际 finValue 显示。
- **finValue 为空字符串**：当作 0 处理，进度 0%。
- **对话框打开期间用户在主页打卡**：对话框为打开瞬间的进度快照，不自动实时刷新；关闭后重新打开反映最新进度。
- **今天恰好是周历第一屏默认选中天**：直接点击即弹框，无需先翻页。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 在用户点击周历中"今天且当前已选中"的日历图标时，弹出居中自定义对话框。
- **FR-002**: 系统 MUST 仅当被点击的是"今天"且已选中时弹框；点击非今天图标仅切换日期选中，不弹框；点击今天但未选中时先切换选中、不弹框，需再次点击已选中的今天图标才弹框。
- **FR-003**: 对话框 MUST 列出当天所有 isOpen=true 的任务条目，按任务定义顺序排列。
- **FR-004**: 每个任务条目 MUST 显示：任务图标、任务名、横向进度条、数值文字。
- **FR-005**: 布尔型任务（step===0，如早起 taskId=0、早睡 taskId=5）进度 MUST 为 0%（未完成）或 100%（已完成），完成时文字显示"已完成"。
- **FR-006**: 次数型任务（step>0，如喝水/吃苹果/微笑/刷牙）进度 MUST = Number(finValue)/Number(targetValue)*100%（封顶 100%），文字显示"finValue/targetValue unit"，完成（isDone 或比例达100%）时文字显示"已完成"。
- **FR-007**: 对话框 MUST 提供关闭方式（点击外部空白或关闭按钮），关闭后主页交互恢复。
- **FR-008**: 对话框打开时读取当天进度快照，不阻塞后台打卡；关闭重开反映最新数据。
- **FR-009**: 当天无已开启任务时，对话框 MUST 显示空状态提示而非空白或报错。

### Key Entities *(include if feature involves data)*

- **TaskInfo**: 任务定义实体，含 taskId、isOpen、targetValue、finValue、isDone；isOpen 用于判断当天是否开启（决定是否进对话框列表）。
- **DayTaskInfo**: 当天任务实例实体，含 date、taskId、targetValue、finValue、isDone；作为对话框每个任务进度的数据源（当天的 finValue/isDone）。
- **TaskBaseInfo（taskBaseInfoList）**: 任务元信息实体，含 name（任务名）、icon/资源、unit（单位）、step（步进值，0=布尔型一次性，>0=次数型累加）；step 用于区分两类任务进度算法，name/unit/icon 用于条目展示。
- **DayInfo**: 当天汇总实体；周历图标 getProgressImg() 已基于 finTaskNum/targetTaskNum 展示进度图标，本功能复用"今天是否选中"的判断逻辑。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户点击今天已选中的日历图标后，1 秒内弹出进度对话框。
- **SC-002**: 对话框准确展示当天每个开启任务的进度：布尔型仅 0%/100% 两态，次数型按完成比例计算且数值文字与进度条一致，准确率 100%。
- **SC-003**: 点击非今天图标不弹框、仅切换日期；点击今天未选中时先切换不弹框——非今天场景 0 次误弹框。
- **SC-004**: 对话框可正常关闭，关闭后主页日历切换/再次打开对话框等交互 100% 恢复正常。
- **SC-005**: 当天无开启任务时对话框显示空状态提示，不报错、不空白。

## Assumptions

- 弹框采用与现有 TaskClockCustomDialog 一致的居中自定义对话框实现方式（CustomDialogController 或 UIContext.openCustomDialog），视觉风格统一。
- 进度条使用 ArkUI Progress 组件（线性 Linear），颜色采用品牌色（sys.color.brand）。
- "今天"判定复用 HomeStore.checkCurrentDay()（现有 TaskListComponent 已用于判断是否可打卡），保证口径一致。
- 任务进度数据复用现有 DayTaskInfoApi 查询当天 dayTaskInfo，并搭配 taskBaseInfoList 获取 name/unit/icon/step。
- 布尔型任务识别采用 step===0（与 pickerType===PickerType.TIME 等价，实现选其一即可）。
- 对话框为快照模式：打开瞬间读取进度，打开后不随主页打卡实时刷新，关闭重开反映最新。
- 数值文字复用现有 targetValueBuilder 的展示约定（finValue || '--'、' / targetValue '、unit，完成显示"已完成"资源串），保证与任务列表一致。
- 次数型百分比显示精度（整数 vs 一位小数）在 Phase 2 设计期确定，不影响范围。

## Open Questions

- 次数型任务进度百分比的显示精度（整数如 33% 还是保留一位小数如 33.3%）待设计期确定；不影响功能范围与数据。
