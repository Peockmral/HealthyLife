# 用户登录注册功能实现计划

## Goal

在 HealthyLife 应用中实现完整的用户登录/注册流程。未登录状态打开应用→显示登录页；已登录状态打开应用→直接进入主页。登录页可跳转注册页，注册完成返回登录页。Mine 页面可退出登录返回登录页。

## Scope / Non-goals

**包含**：
- 登录页面（用户名+密码输入，登录按钮，注册按钮）
- 注册页面（用户名+密码+确认密码，注册按钮，登录按钮）
- 登录状态持久化（Preferences）
- 注册用户存储（Preferences JSON string）
- Mine 页面退出登录按钮
- 应用启动时的登录状态判断

**不包含**：
- 网络请求/后端 API（纯本地存储）
- 密码加密/哈希（明文存储，简单本地应用）
- 第三方登录（微信/华为账号等）
- 密码找回/修改功能
- Token/Session 过期机制

## Current State And Constraints

- **EntryAbility** 在 `onWindowStageCreate` 中按序 await 初始化 RDB + Preferences，最后 `loadContent('pages/Index')`
- **Index.ets** 是 `@Entry` 页面，仅渲染 `HealthyLifePage()`
- **HealthyLifePage** 有4个 `@Provide`（achievementStore, pathStack, homeStore, profileStore），`aboutToAppear` 异步初始化它们
- **Navigation(pageStack) { Tabs {...} }** 架构——Tabs 是根内容，NavDestination 页面通过 pushPathByName 叠加
- **PreferencesUtils** 单例 store，支持 `putPreference(key, value)` / `getPreference(key): Promise<string>`
- **ProfileStore** 模式：`@Observed` class + `initXxx()` 读取 Preferences + `updateXxx()` 写入并返回新实例
- ArkTS 约束：无 `any`/`unknown`/`as unknown`，无解构，无对象字面量做类型，`Record<K,V>` 用引号键
- `@Provide`/`@Consume` 双向同步——子组件修改 `@Consume` 会传播回 `@Provide`

## Design

### 核心架构：条件渲染切换登录页/主页

在 `HealthyLifePage` 中添加 `@Provide isLoggedIn: boolean`，`aboutToAppear` 从 Preferences 读取登录状态。`build()` 中条件渲染：

```
Navigation(this.pageStack) {
  if (this.isLoggedIn) {
    Tabs { HomeComponent, MineComponent }   // 已登录→主页
  } else {
    LoginComponent()                        // 未登录→登录页
  }
}
```

- **LoginComponent** 是内联组件（非 NavDestination），作为未登录时的根内容
- **RegisterComponent** 是 NavDestination，通过 `pageStack.pushPathByName` 叠加在 LoginComponent 之上
- 登录成功 → `this.isLoggedIn = true`（`@Consume`→`@Provide` 双向传播）→根内容切换为 Tabs
- 退出登录 → `this.pageStack.clear()` + `this.isLoggedIn = false` → 根内容切换为 LoginComponent

### D1: 登录状态存储

**决策**：Preferences 键值对 `IS_LOGGED_IN = 'isLoggedIn'`（'true'/'false'），在 `PreferencesUtils.getPreferencesFromStorage` 初始化默认值 'false'。

**理由**：与现有 ProfileStore、AchievementStore 使用同一 Preferences store，无需新建 store。

### D2: 注册用户存储

**决策**：Preferences 键 `REGISTERED_USERS = 'registeredUsers'`，值为 JSON string `'{"user1":"pass1","user2":"pass2"}'`，默认 `'{}'`。

**理由**：支持多用户注册，无需新建 RDB 表。`JSON.parse`/`JSON.stringify` 已在项目中使用（AchievementStore.ets）。ArkTS 中用 `Record<string, string>` 类型。

### D3: AuthUtils 工具类

**决策**：新建 `AuthUtils` 静态类（放在 `commons/common`），提供注册/验证/登录状态管理函数。

**理由**：与 `PreferencesUtils`、`RequestAuthorization` 同为 `commons` 中的静态工具类，被 features 模块导入。

### D4: 页面 UI 设计

**LoginComponent**（内联组件，非 NavDestination）：
- 标题 "登录"
- TextInput 用户名（placeholder "请输入用户名"）
- TextInput 密码（InputType.Password，placeholder "请输入密码"）
- Button "登录"（主色按钮，验证后设 isLoggedIn=true）
- Text "注册"（文字按钮/链接样式，点击 pushPathByName RegisterComponent）
- 无返回按钮（根页面）

**RegisterComponent**（NavDestination）：
- 返回按钮 + 标题 "注册"
- TextInput 用户名
- TextInput 密码
- TextInput 确认密码
- Button "注册"（验证后注册用户，pop 返回登录页）
- Text "登录"（文字按钮，pop 返回登录页）

### D5: Mine 页面退出登录

**决策**：在 Mine 页面 Achievement 行下方添加独立 `Button`（红色文字 "退出登录"），点击后：clear pageStack → putPreference(IS_LOGGED_IN, 'false') → this.isLoggedIn = false。

**理由**：退出登录是危险操作，独立按钮比行内选项更醒目。红色文字区分。

## Execution Sequence

### Step 1: 基础设施

1. `CommonConstants.ets` — 添加 `IS_LOGGED_IN = 'isLoggedIn'` 和 `REGISTERED_USERS = 'registeredUsers'` 常量
2. `PreferencesUtils.ets` — 在 `getPreferencesFromStorage` 中添加 `IS_LOGGED_IN` 默认 'false'，`REGISTERED_USERS` 默认 '{}' 的初始化
3. 3个 `string.json` — 添加字符串：login_title(登录/Login), register_title(注册/Register), username_label(用户名/Username), password_label(密码/Password), confirm_password_label(确认密码/Confirm Password), login_button(登录/Login), register_button(注册/Register), logout_button(退出登录/Logout), login_failed(用户名或密码错误/Invalid username or password), password_mismatch(两次密码不一致/Passwords do not match), user_exists(用户名已存在/Username already exists), register_success(注册成功/Registration successful), username_empty(请输入用户名/Please enter username), password_empty(请输入密码/Please enter password)

### Step 2: AuthUtils

4. `AuthUtils.ets` — 实现静态类：
   - `getRegisteredUsers(): Promise<Record<string, string>>` — 读取 REGISTERED_USERS JSON，parse 为 Record
   - `registerUser(username, password): Promise<boolean>` — 读取现有用户，检查用户名是否存在，不存在则添加，写回 Preferences，返回 true/false
   - `validateLogin(username, password): Promise<boolean>` — 读取用户，检查用户名+密码匹配
   - `isLoggedIn(): Promise<boolean>` — 读取 IS_LOGGED_IN === 'true'
   - `setLoggedIn(value: boolean): Promise<void>` — 写入 IS_LOGGED_IN

### Step 3: LoginComponent

5. `LoginComponent.ets` — 创建登录页面（内联组件，非 NavDestination）：
   - `@Consume('pathStack') pageStack` + `@Consume isLoggedIn: boolean`
   - `@State username: string = ''` + `@State password: string = ''`
   - UI：标题 Text + TextInput 用户名 + TextInput 密码 + Button 登录 + Text 注册（可点击）
   - 登录 onClick：验证用户名密码非空 → `AuthUtils.validateLogin()` → 成功则 `AuthUtils.setLoggedIn(true)` + `this.isLoggedIn = true`
   - 失败 → showToast(login_failed)
   - 注册 onClick：`this.pageStack.pushPathByName('RegisterComponent', '')`

### Step 4: RegisterComponent

6. `RegisterComponent.ets` — 创建注册页面（NavDestination）：
   - `@Consume('pathStack') pageStack`
   - `@State username/password/confirmPassword: string = ''`
   - UI：返回按钮 + 标题 + 3个 TextInput + Button 注册 + Text 登录（可点击）
   - 注册 onClick：验证非空 → 验证两密码一致 → `AuthUtils.registerUser()` → 成功 showToast + `pageStack.pop()` → 失败 showToast(user_exists)
   - 登录 onClick：`pageStack.pop()`
7. `router_map.json` — 添加 RegisterComponent 路由

### Step 5: HealthyLifePage 条件渲染

8. `HealthyLifePage.ets` — 添加 `@Provide isLoggedIn: boolean = false`，在 `aboutToAppear` 中 `this.isLoggedIn = await AuthUtils.isLoggedIn()`，`build()` 中条件渲染

### Step 6: MineComponent 退出登录

9. `MineComponent.ets` — 添加 `@Consume isLoggedIn: boolean`，Achievement 行下方添加退出登录 Button（红色），onClick：`this.pageStack.clear()` + `AuthUtils.setLoggedIn(false)` + `this.isLoggedIn = false`

### Step 7: 构建验证

10. `arkts_check` 所有新/改文件
11. `build_project` 编译验证

## Risks And Compatibility

- **JSON.parse 类型安全**：ArkTS 严格模式要求显式类型。`JSON.parse(str) as Record<string, string>` — 需确保 JSON.parse 的返回值正确断言。已有项目中 `AchievementStore.ets` 使用 `JSON.parse(str) as boolean` 先例。
- **条件渲染时序**：`aboutToAppear` 是 async，首次渲染 `isLoggedIn=false` 可能短暂闪现登录页。与现有 ProfileStore/AchievementStore 异步初始化模式一致，可接受。
- **pageStack.clear()**：退出登录时需清空 NavPathStack，防止已 push 的子页面残留。`NavPathStack.clear()` 是标准 API。
- **LoginComponent 不在 router_map.json 中**：它是内联条件渲染组件，不是 pushPathByName 推入的 NavDestination，无需注册路由。只有 RegisterComponent 需要注册。
- **密码明文存储**：简单本地健康应用，不涉及网络传输，明文存储可接受。如需增强安全性可后续添加哈希。
